nmap tutorial

 nmap is an open-source tool used for network discovery and security auditing. it can be used for discovering which systems are up and what services are runnning on these systems.

 steps in typical nmap scan:

  • enumerating the targets
  • discovering the live hosts
  • reverse-DNS lookup
  • scanning the ports
  • detecting the OS
  • traceroute
  • using scripts
  • writing the output

live host discovery

 discovering the online systems is essential, as trying to port-scan offline systems consumes a lot of time and creates unnecessary noise on the network. by default, nmap discovers the live hosts and then port-scans the live hosts only. to disable port-scanning, we use the -sn flag.

 ARP(address resolution protocol) is a network protocol used to map an IP address to its corresponding MAC address within a local network, so that communication can take place. it only operates on the same subnet.

 ICMP(internet control message protocol) is a network protocol used for sending error messages and operational information, mainly to diagnose network communication issues(e. g. with the ping command).

 TCP(transmission control protocol) is a connection-oriented protocol that ensures reliable communication between applications by using acknowledgments, retransmissions and error checking.

 UDP(user datagram protocol) is a connectionless protocol that sends data without guaranteeing delivery, making it faster but less reliable, often used for streaming, gaming or VoIP.

reverse-DNS lookup

 nmap's default behaviour is to use reverse-DNS lookup. this can be helpful, but time consuming. you can use -n flag to skip it or -R flag to force it even on offline hosts.

enumerating targets

 general layout of Nmap: nmap [options] [targets]

 ways to specify targets:

  • subnet: 192.168.1.0/24 will scan the whole subnet(256IPs)
  • range: 192.168.1.0-4 will scan 5 Ips: 192.168.1.0, 192.168.1.1 ... 192.168.1.4
  • files : nmap -iL list.txt will scan all IPs in list.txt file
  • list: 192.168.1.0, 192.168.0.9, 192.168.0.45 will scan 4 IPs

 to list all the IPs that are going to be scanned you can use the -sL flag :

nmap -sL 192.168.1.0/30

host discovery using ARP

 ARP scan is only possible if you are on the same subnet as your targets. on ethernet and wifi, you have to know the MAC addresses of the target systems, in order to communicate with them. to obtain it, the OS sends an ARP query. a host that replies to the ARP query is up. we expect to see many ARP queries when scanning a local network.

 we can force Nmap to use ARP for host discovery by using the -PR flag.

host dicovery using ICMP

 we can ping all IPs on the network with ICMP echo requests(ICMP Type 8) and see who replies with ICMP ping reply(ICMP Type 0). it is straightforward, but many firewalls and modern Windows versions block ICMP pings by default. ARP query will precede the ICMP request, if you and the target both are on the same subnet.

 to use ICMP echo requests to discover live hosts, use -PE flag. remember that many firewalls block ICMP.

 we don't expect to learn MAC addresses, unless we're on the same subnet as the target.

example: nmap -PE -sn 192.168.1.0/24

 because ICMP echo requests tend to be blocked, you can use ICMP timestamp request(ICMP Type 13) and see who replies with Timestamp reply(ICMP Type 14). adding -PP flag tells Nmap to do that.

 we can also include -PM flag to use Address mask queries(ICMP Type 17) and see who replies with Address mask reply(ICMP Type 18).

Host discovery using TCP/UDP

 We can send a packet with SYN(Synchronise) flag to a TCP port, 80 by default. An open port should reply with SYN/ACK(Acknowledge), while a closed one should reply with RST(Reset). We do not care what response we receive, only whether or not we receive one.

 To use TCP SYN ping, we specify the -PS flag. Privileged users can send TCP SYN packets and don't need to complete the 3-way handshake. Unprivileged users have to.

 To use TCP ACK ping, you need to be privileged user. It works by sending an ACK flag to a TCP port(80 by default). The port should reply with RST flag, regardless of the state it has. We can tell Nmap to use this type of scan by specifying the -PA flag.

 We can use UDP to discover if the host is online. Sending a UDP packet to an open port is not expected to lead to a reply, but sending to a closed port results in a ICMP port unreachable reply. Nmap sends UDP packets to high, most likely unused ports. We can use the -PU flag to tell it to use UDP scan.


Summary

 Summary of the flags used:

  • -PR - ARP scan
  • -PE - ICMP echo scan
  • -PP - ICMP timestamp scan
  • -PM - ICMP address mask scan
  • -PS - TCP SYN Ping scan
  • -PA - TCP ACK Ping scan
  • -PU - UDP Ping scan
  • -n - no DNS lookup
  • -R - reverse-DNS lookup for all hosts
  • -sn - host discovery only


Basic port scans

TCP and UDP ports

 Like an IP address identifies a host on a netowork, a TCP or UDP port identifies a networking service on a host. Ports are usually linked to specific services, for example HTTP on TCP port 80, SSH on TCP port 22. No more than one service can listen on any port.

 A port can be open, which means a service is listening on it, or closed, which means no service is listening on the port. In practical situations, a firewall or other security measure might block packets, so Nmap recognises 6 port states:

  • Open: a service is listening on the port.
  • Closed: no service is listening on the port, but it is accesible(it is reachable and not blocked by a firewall or another security measure).
  • Filtered: Nmap cannot determine if the port is open or closed, because it is inaccesible. Usually due to a firewall preventing Nmap from reaching the target or the target from reaching Nmap.
  • Unfiltered: Nmap cannot determine if the port is open or closed, but it is accesible.
  • Open|Filtered: Nmap cannot determine if the port is open or filtered.
  • Closed|Filtered: Nmap cannot determine if the port is closed or filtered.

TCP flags

 A TCP segment can have one or many TCP flags, located in its header. They can be either set or unset:

  • URG - The incoming data is urgent, the segment will be processed immediately, without waiting for others.
  • ACK - Used to acknowledge the receipt of a TCP segment.
  • PSH - Push flag asking TCP to pass the data to the application promptly.
  • RST - Reset flag used to reset the connection.
  • SYN - Synchronise flag used to initiate 3-way handshake and synchronise sequence numbers with host.
  • FIN - The sender has no more data to send.

TCP connect scan

 The TCP connect scan works by completing the 3-way handshake. After, it sends a RST/ACK flag to tear it. If you are unprivileged user, TCP connect scan is the only way to discover open TCP ports. To use it, we can include the -sT flag. The -F flag decreases the number of ports scanned, from 1000 to 100 most common ports.

TCP SYN scan

 SYN scan does not complete the 3-way handshake. It tears the connection once it receives a response from the target. Because we didn't establish a connection, it is much more stealthier. It can be used with the flag -sS.

UDP scan

 UDP is a connectionless protocol, it does not require any handshake. We cannot guarantee that a service listening on a UDP port will reply, but if a UDP packet is sent to a closed port, it will return an ICMP port unreachable error(ICMP Type 3, Code 3). You can use this flag with the -sU flag, and it can be combined with other TCP scans.


Choosing ports, speed and parallelization

 By default, Nmap scans the most common(top) 1000 ports. You can use:

  • -p21,23,8080 to scan ports 21,23,8080
  • -p1-10 to scan 10 ports
  • -p- to scan all 65 thousand ports
  • -F to scan most common 100 ports
  • --top-ports 50 to scan 50 most common ports

 You can control the speed using -T0-5. Different speeds:

  • 0-paranoid
  • 1-sneaky(used in real world scenarious)
  • 2-polite
  • 3-normal
  • 4-aggresive(used in CTFs)
  • 5-insane

 You can control packet rate with --min-rate [number] and --max-rate [number]. You can control probing parallelization with --min-parallelism [numprobes] and --max-probes [numprobes].

Summary

 Summary of flags used:

  • -sT - TCP connect scan
  • -sS - TCP SYN scan
  • -sU - UDP scan
  • -p- - scan all ports
  • -F - scan most common 100 ports
  • -p1-10 - scan all ports from 1 to 10
  • -T<0-5> - 0=slowest 5=fastest

Advanced port scans

 Security researchers and hackers combined different TCP flags in segments and started to experiment. They wanted to find out what happens if we send a TCP packet with some flags set, when it is not part of any connection.

Different scan types

  1. Null scan: with -sN flag, has no TCP flags. A null packet sent to an open port doesn't generate any response. Sent to a closed one, returns a RST/ACK response. It cannot determine whether the port is open or closed.

  2. FIN scan: with -sF flag, has only FIN flag set. A FIN packet sent to on open port doesn't generate any response. Sent to a closed one, returns a TCP/ACK packet. It cannot determine whether the port is open or filtered.

  3. Xmas scan: with -sX has FIN, PSH and URG flags set. Sent to an open port, it does not generate a response. If sent to a closed one, it returns a RST/ACK packet. It cannot determine wether the port is open or filtered.

  4. Maimon scan: with -sM has FIN and ACK flags set. It is mostly useless.
  5. TCP ACK scan: with -sA flag, has ACK flag set. A port replies with RST regardless of its state. It is only usefull to detect firewall rules, which ports are blocked and which ports are not. It is ineffective against targets without firewalls.
  6. Window scan: with -sW flag, has ACK flag set. Similar to TCP ACK scan, but it examines the window field of the RST packets returned. It is only usefull to detect firewall rules, which ports are blocked and which ports are not. It is ineffective against targets without firewalls.
  7. Custom scan: with --scanflags URGACKPSHRSTSYNFIN. You can select the flags you want to set.

Spoofing and decoys

 In some network setups, you will be able to scan a target system with a spoofed IP address and even MAC address. You can use this only when you can guarantee to capture the response. Scanning on random networks using a spoofed IP address would return unreliable results, unless you have the response routed to you.

 The attacker needs to monitor the network traffic to analyse replies. Steps for scanning with spoofed IPs:

  1. Attacker sends a packet with a spoofed source IP address to the target machine.
  2. The target machine replies to the spoofed IP address as the destination.
  3. Attacker captures the replies to figure out open ports.

 In general, it is expected to specify the network interface with -e NET-INTERFACE and explicitly disable ping scan with -Pn flag. The attack is useless if the attacker doesn't monitor the network for responses.

 You can also spoof MAC addresses with --spoof-mac SPOOFED_MAC, but only if you are on the same subnet.

 Spoofing only works in minimal number of cases, where certain conditions are met. The attacker might also use decoys to make it harder to be pinpointed. The concept is to make the scan appear to be coming from many IP addresses, so the attacker's IP is lost among them. You can do this with :

-D IPs. You can use ME to indicate your IP address or RND for random IP addresses.

Example: nmap -D 192.168.1.12,192.168.1.67,ME,RND


Fragmented packets

 You can fragment packets to make it harder for IDS or Firewall to detect them. You can use -f to divide the IP data into 8 bytes or less. It is stackable, so -ff divides it into 16. You can increase the size of the packets with --data-length NUM.

 You can use --source-port PORT flag to specify the source port.

Verbosity

 You can adjust the verbosity of the output with -v flag. It is stackable up to -vvvvv. --reason flag displays the reason why Nmap classified a port's state. You can also use -d to print debugging details. It is stackable too.

Post port scans

 In this article, we will be focusing on how Nmap can be used to:

  • Detect versions of running services
  • Detect the OS based on any signs
  • Run Nmap's traceroute
  • Run Nmap scripts
  • Save results in various formats

Service detection, OS detection and traceroute. Saving the results.

 Including -sV flag will collect and determine services and version information for the open ports. The level of information can be specified with --version-intensity LEVEL where LEVEL can take a value between 0 and 9. 0 is least amount of details, 9 the most. Also, using -sV forces Nmap to complete the 3-way handshake

 Nmap can attempt to determine the host OS, when given the -O flag. It can also attempt to find routes between you and the target, using --traceroute.

 Nmap can save the output in 3 different formats: normal format(using -oA flag), grepable format(using -oG flag) or XML format(using -oX flag).

 -oA flag can be used to save in all formats. Also, you have to specify the filename, along with these flags. For example, adding -oN output to the command will save it as a file named output.

Nmap scripting engine

 A script is a sequence of instructions that can be executed by a computer or programming language, typically used to automate tasks that would otherwise be performed manually. In Nmap, a script can reveal additional information about the host or ports and services running on it. These can be used to exploit vulnerabilities or help compromise the system.

 The default Nmap installation contains about 600 scripts, located in the /usr/share/nmap/scripts. Script categories:

  • auth: authentication related scripts
  • broadcast: discover hosts by sending broadcast messages
  • brute: performs brute-force passwords auditing against logins
  • default: default scripts, same as --script=default
  • discovery: retrieve accesible information, such as database tables and DNS names
  • dos: detects servers vulnerable to Denial of Service(DoS)
  • exploit: attempt to exploit various vulnerable services
  • external: checks using a third-party service, such as Geoplugin and Virustotal
  • fuzzer: launch fuzzing attacks
  • intrusive: intrusive scripts such as brute-force attacks and exploitation
  • malware: scans for backdoors
  • safe: safe scripts that won't crash the target
  • version: retrieve service versions
  • vuln: check for vulnerabilities or exploit vulnerable services

 You can choose a script using --script SCRIPT-NAME. You can use more scripts with wildcards.
