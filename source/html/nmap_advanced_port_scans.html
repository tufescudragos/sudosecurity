nmap advanced port scans

security researchers combined different TCP flags in packets and sent them to open/closed ports to determine the responses. they wanted to find out what happens when we send a packet with some TCP flags set, when there isn't any connection established. they concluded and created new scan types

null scan
null scan can be triggered with -sN
it has no TCP flags set. sent to an open port, it does not generate response. sent to a closed one, returns RST/ACK packet
can't decide whether the port is open or filtered




FIN scan
FIN scan can be triggered with -sF
it has only FIN flag set. sent to an open port, it does not generate response. sent to a closed one, returns RST/ACK packet
can't decide whether the port is open or filtered




Xmas scan
Xmas scan can be triggered with -sX
it FIN, PSH, URG flags set. sent to an open port, it does not generate response. sent to a closed one, returns RST/ACK packet
can't decide whether the port is open or filtered




maimon scan
with -sM. it is mostly useless. has FIN and ACK set
custom scan
with --scanflags URGACKPSHRSTSYNFIN. you can select the flags you want to set

verbosity

for a more detailed output, the flags -v and even -vv can be used. v stands for verbosity.

providing the --reason option provides the reason why nmap concluded that a system is up or a particular port open.

-d and -dd provide debugging details. they are very large in size.

summary

  • -sN - TCP null scan
  • -sF - TCP FIN scan
  • -sX - TCP Xmas scan
  • -sM - TCP Maimon scan
  • --scanflags URGACKPSHRSTSYNFIN - custom scan
  • -v - verbose
  • --reason - show reason
  • -d - debugging
